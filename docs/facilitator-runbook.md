## 느린 참가자 처리

느린 참가자는 완성도보다 빈칸 없는 초안을 목표로 합니다. 숫자가 없으면 `[가정]`이라고 표시하게 하고 다음 단계로 넘깁니다.

## 빠른 참가자 처리

빠른 참가자(쉽다/지루하다고 느끼는 분)는 아래 심화 루프로 보냅니다.

1. 합성 소비자 패널 5명 생성 (`korean-synthetic-consumer`)
2. 메시지 테스트 · 반대 논리 맵
3. 실제 인터뷰 질문 7개 생성
4. `evidence-research`로 핵심 숫자(TAM·가격·CAC) 근거 1~2개 보강
5. `vc-red-team-review` 심화 + 최종 피치 수정
6. **MVP를 한 단계 더**: `startup-cto-roadmap`로 화면 흐름·기술 결정까지, 가능하면 작은 프로토타입/랜딩 시도
7. **실무 위임 연습**: 메인 가이드의 "창업자에게 유용한 Harness 아이디어"/gstack 역할 위임 체크리스트로 내 업무 자동화 1개 설계

## 설문 반영 — Day 2 운영 조정

5/28 만족도 설문(만족 4.6/5·NPS +43, 전원 후속 참여 의향)에서 나온 개선점을 Day 2에 반영합니다.

- **용어 장벽** ("줄임말·용어가 어렵다") → 줄임말이 나올 때마다 한 번 풀어 설명하고, `docs/용어집.md`를 화면에 띄워 둔다. 새 용어가 나오면 즉석에서 용어집에 추가.
- **"실제로 만들어지는 과정·실무 활용" 욕구** → Day 2는 개발 로드맵·실행 단계임을 앞에서 강조하고, "에이전트로 업무를 어떻게 위임·자동화하는지"를 작은 데모로 보여준다.
- **분야별 협업 매칭** → 피어 리뷰(15:00)를 **비슷한 분야끼리 3인 1조**(예: 헬스케어 / 플랫폼·마켓 / B2B·데이터)로 묶어 시너지를 낸다. 세션 끝에 5분간 연락처·milgram 프로필 교류 시간.
- **페이스** ("조금 느렸다") → 빠른 참가자는 위 심화 트랙으로 분리해 전체 진도를 끌지 않게 한다.
