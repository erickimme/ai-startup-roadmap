# 샘플 런 — 건강이음(HealthBridge)

> 코치(orchestrator + 스킬)를 **full 모드**로 돌린 예시입니다. 한 참가예정자의 실제 제출 아이디어
> ("AI로 의료 정보 접근성이 낮은 사람들이 건강 정보를 이해·활용하는 플랫폼")를 P0(비영어권 이주민)로
> 좁혀 9종 산출물을 만든 결과이며, `roadmap/` 폴더가 단계마다 어떻게 쌓이는지 보여줍니다.
>
> 학습용 예시이므로 숫자는 대부분 `[가정]`입니다. 실제 진행 시에는 인터뷰·근거로 대체하세요.

## 산출물 목차
1. [아이디어 정의서](01_아이디어정의서.md) — 넓은 아이디어를 P0로 정제, 엘리베이터 피치
2. [BMC](02_BMC.md) — 9블록 + TAM/SAM/SOM + unit economics (B2G 모델 발견)
3. [GTM](03_GTM.md) — 다문화센터·지자체 통한 기관 진입
4. [세일즈 플랜](04_세일즈플랜.md) — B2G 라이선스 + 그랜트(사용자 무료)
5. [개발 로드맵](05_개발로드맵.md) — RAG 기반 MVP, 민감정보·안전가드
6. [HR 플랜](06_HR플랜.md) — 의료 자문(감수)을 초기 핵심 역할로
7. [실행 타임라인](07_실행타임라인.md) — OKR + 리스크(오정보=치명)

**심화 (full 모드 / 빠른 참가자)**
10. [근거 리서치](10_근거리서치.md) — 숫자에 출처·신뢰도 부여 (evidence-research)
11. [합성 소비자 패널](11_합성소비자패널.md) — 페르소나·메시지 테스트·반대 논리 (korean-synthetic-consumer)

## 이 샘플이 보여주는 코치의 행동
- 넓은 아이디어를 **질문으로 좁힘**(Stage 1)
- 단계마다 **가장 약한 가정**을 명시하고 `[가정]` 표기
- 도메인 특성(의료 윤리·오정보·개인정보)을 산출물에 반영
- 레드팀이 약점을 공격 → 최종본에서 수정
