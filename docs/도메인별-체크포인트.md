# 도메인별 검증 체크포인트

아이디어 유형마다 **가장 먼저 깨지는 가정**과 **봐야 할 지표**가 다르다. 단계 진행 중 아래에서 해당 유형을 찾아 "지금 당장 검증할 것"과 "핵심 KPI"에 반영한다. 여러 유형에 걸치면 둘 다 본다.

> 사용법: orchestrator가 아이디어를 받으면 먼저 아래 유형 1개 이상을 고른다. BMC·세일즈·레드팀 단계에서 해당 유형의 "반드시 검증"을 질문/지표로 넣는다.

---

## 1. 양면 마켓플레이스 · 매칭 (수요자 ↔ 공급자)
예: 포토 매칭, 굿즈 셀러 플랫폼, 도메인 큐레이션, 중개 거래

- **가장 먼저 깨지는 가정:** 닭-달걀(공급·수요 동시 확보), 양쪽이 플랫폼을 거칠 이유(탈중개 방지).
- **반드시 검증:** 한쪽을 **수동으로 먼저** 채워 매칭 1건을 만들 수 있는가 · 공급자가 단가/시간에 응할 동기 · 거래가 플랫폼 밖(DM·직거래)으로 새지 않는가 · take rate가 단위 경제를 버티는가.
- **핵심 지표:** 첫 매칭까지 걸린 시간, 매칭 성사율, take rate, 공급자 응답률/재참여율, GMV·건당 기여이익.
- **흔한 함정:** 단가가 낮아 수수료만으로 운영비를 못 넘김(→ 객단가·반복·B2B 보조매출 필요), "SNS·검색으로 직접 하면 되는데 왜?"에 답 못 함.

## 2. B2B 생산성 · 현장 도구 (SaaS/내부 도입)
예: 업무 자동화, 현장 엔지니어용 도구, 운영 프로세스

- **가장 먼저 깨지는 가정:** 개인이 좋아해도 **회사가 도입**한다(보안 승인·결제 권한·파일 반출 규정).
- **반드시 검증:** 개인 결제 가능 여부 vs 회사 승인 경로 · 보안/규정으로 막히는 지점 · 기존 작업 대비 절감 시간(분 단위) · 파일럿 → 유료 계약 전환.
- **핵심 지표:** 절감 시간/비용, 파일럿 수, 파일럿→유료 전환율, 좌석당 ARPU, 도입 사이클 길이, 이탈률.
- **흔한 함정:** "월 ○○원이면 쓸 의향"을 도입 확정으로 착각(권한·보안 미확인), 팀플랜을 너무 일찍 설계.

## 3. 딥테크 · 생성형 (기술 실현이 핵심 리스크)
예: 생성형 3D/CAD, 모델·파이프라인이 결과 품질을 좌우하는 제품

- **가장 먼저 깨지는 가정:** 데모는 되는데 **실사용 입력에서 성공률**이 유지된다.
- **반드시 검증:** 좁은 1개 유형에서 생성/처리 **성공률 목표치**(예: 70%) · 결과물이 실제 후처리·사용 가능한 수준인가 · 실패 케이스 분류 · 안전/격리 실행.
- **핵심 지표:** 첫 시도 성공률, 결과물 사용/편집 가능률, 실패 유형 상위 3개, 생성당 비용(연산).
- **흔한 함정:** 범위를 일찍 넓힘(성공률 검증 전 부품/케이스 확장), 화면(UI)에 시간 쓰고 핵심 성공률을 안 봄.

## 4. 소비자 앱 · 크리에이터 (B2C, 저단가·반복)
예: 크리에이터 도구, 취미·여행·콘텐츠 앱

- **가장 먼저 깨지는 가정:** "재미있다/좋아요"가 **결제·반복 사용**으로 이어진다.
- **반드시 검증:** 무료→유료 전환의 명분(횟수 제한·핵심 기능) · 리텐션(2회차 행동) · CAC vs 낮은 객단가 · 구매 의향을 **실제 결제/예약금**으로 확인.
- **핵심 지표:** 활성화율, 2회차 사용률, 무료→유료 전환율, CAC·LTV(LTV/CAC<3 위험), 추천/바이럴 계수.
- **흔한 함정:** 좋아요·조회수를 수요로 오해, 구독을 매출 발생 전부터 기대(저단가·1회성 소비 가능성).

## 5. 규제 민감 (의료·제약·금융·법률·아동·결제·정산)
예: 의료정보, 의약품, 핀테크 거래, 리뷰/중개 정산, 아동 대상

- **가장 먼저 깨지는 가정:** 제품이 좋아도 **규제·책임·인허가**가 출시를 막는다.
- **반드시 검증:** 어떤 법·인허가·자격이 걸리는가(의료기기·전금법·개인정보·광고규제 등) · 책임 범위와 면책 · 무엇을 "하지 않는다"로 빼서 규제를 피하는가 · 정산/분쟁 구조.
- **핵심 지표:** 규제 게이트 통과 여부, 자문 비용, 책임/분쟁 발생률, (해당 시) 임상·인증 일정.
- **흔한 함정:** 규제를 "나중 문제"로 미룸, 합성/AI 결과를 검증 완료처럼 표기. 산출물에 `[법률 검토 필요]`를 명시.

## 6. 커리어 · 교육 · 매칭형 진입 도구
예: 포트폴리오·취업·진로, 역량 매핑

- **가장 먼저 깨지는 가정:** 무료로 결과물은 받지만 **돈을 낼 만큼** 강한 필요가 있다. 일반 생성형 AI로 충분하지 않다고 느낀다.
- **반드시 검증:** 일반 ChatGPT 대비 차별점(검증된 사람의 리뷰·도메인 특화 등) · 성과 지표 정의(완성? 합격? 제출?) · 결과물의 실제 제출/활용 여부.
- **핵심 지표:** 결과물 완성률, 실제 제출/활용률, 유료 전환율, 추천 의향.
- **흔한 함정:** "AI 생성" 자체를 장기 독점 강점으로 과장, 성공 정의가 모호해 측정 불가.

---

## 공통 원칙 (모든 유형)
- 외부 숫자(시장·가격·CAC)는 **출처(URL)** 먼저, 못 찾으면 `[가정]`. (`evidence-research`)
- 각 단계 "가장 약한 가정 → 왜 약한가 → 깨는 방법(검증 행동)"을 남긴다.
- 검증은 설문이 아니라 **실제 행동**(결제·예약금·계약 의향)으로 확인한다.
