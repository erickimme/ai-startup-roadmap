# 디자인 가이드 — "바이브코딩 티 안 나게" 만들기

배포는 빠르게(못생겨도 OK), 디자인은 그 다음. 목표는 **"AI가 찍어낸 것 같지 않은"** 페이지를 만드는 것. 그게 엣지다.

## 2단계 워크플로우
- **Phase 1 — 빠르게 배포(속도):** 핵심 내용·waitlist·흐름이 **라이브에서 되는지 먼저 확인**. 디자인은 신경 쓰지 않는다.
- **Phase 2 — 디자인 올리기(엣지):** 아래 4스텝으로 "AI 티"를 벗긴다.

## Phase 2: 레퍼런스 기반 디자인 4스텝
1. **레퍼런스 훔치기** — [Mobbin](https://mobbin.com) / [Land-book](https://land-book.com) / [Godly](https://godly.website) / 평소 좋아하던 실제 사이트에서 **딱 1개**를 고른다. 스크린샷을 에이전트에 공유(또는 URL·상세 묘사)하고: *"이 레이아웃·여백·타입 굵기·색 절제를 따라 만들어줘. 우리 카피로."*
2. **디자인 토큰부터 고정** — 색(브랜드 1~2색 + 중립 1세트), 폰트(본문 1 + 디스플레이 1), 간격 스케일, 라운드 값. Tailwind theme / `globals.css`에 먼저 박는다. **기본 slate/violet + Geist/Inter를 그대로 두는 게 1순위 AI 티.**
3. **shadcn/ui로 컴포넌트 교체** — `npx shadcn@latest init` 후 button·input·card 등을 토큰에 맞춰 variant 조정. 기본 스타일 그대로 쓰지 말고 라운드·보더·그림자·색을 레퍼런스에 맞춘다.
4. **안티-제네릭 체크리스트 적용 → 재배포 → before/after 비교.**

## ❌ "AI 티" 나는 것 (피하기)
- 가운데 정렬 히어로 + 보라/파랑 **그라데이션 blob** 배경
- lucide 아이콘 **3-카드 피처 그리드** (전부 `rounded-2xl`)
- 이모지 불릿, "✨ Powered by AI" 같은 문구
- 전부 같은 회색 텍스트 · 한 가지 굵기 · 다 가운데 정렬
- 기본 Tailwind 팔레트(slate) + 기본 폰트(Geist/Inter)만

## ✅ 엣지를 만드는 것 (하기)
- **구체 레퍼런스 1개를 정확히 모사** — 여백·정렬·타입 위계·색 사용량까지
- **분명한 타입 위계** — 디스플레이는 크고 굵게, 본문은 차분히, 강조색은 소수만
- **의도된 레이아웃** — 비대칭·왼쪽 정렬·그리드 변주, 진짜 카피(로렘 금지)
- **시그니처 요소 1개** — 실제 제품 목업/스크린샷, 개성 있는 폰트, 대담한 헤드라인 중 하나
- **디테일** — 일관된 간격 리듬, 절제된 보더/그림자, hover·focus 상태, 성공/빈 상태까지

## 레퍼런스를 에이전트에 주는 법 (Codex CLI)
- **이미지 첨부가 되면:** Mobbin/사이트 스크린샷을 붙이고 "이 톤·레이아웃·여백으로 우리 페이지를 다시 짜줘."
- **안 되면(묘사 기반):** 레퍼런스 URL + "이 사이트의 팔레트·폰트·간격·무드를 디자인 토큰으로 뽑아서 우리 카피에 적용해줘."
- 핵심: **"예쁘게 해줘"가 아니라 "이 레퍼런스처럼"** 이라고 말할수록 결과가 덜 제네릭하다.

> 이 과정은 `roadmap/`의 어제 카피(가치 제안·문제/해결)를 유지한 채 **겉모습만 레퍼런스 수준으로** 끌어올리는 것이다. 내용을 새로 지어내지 않는다.
