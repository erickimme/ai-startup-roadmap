# 용어집 — 줄임말·테크 용어 쉽게 풀이

> 수업 중 나오는 용어를 한 줄로 정리했습니다. 모르는 말이 나오면 여기서 찾아보세요. (어려운 용어가 또 나오면 강사·조교에게 알려주세요!)

## 🖥 도구·환경
| 용어 | 쉽게 말하면 |
|---|---|
| **CLI** (Command Line Interface) | 글자로 명령하는 검은 창. 마우스 클릭(GUI) 대신 타이핑으로 시킴 |
| **GUI** (Graphic User Interface) | 마우스로 클릭하는 보통의 화면 (폴더·버튼) |
| **터미널 / PowerShell** | CLI를 입력하는 앱. Mac=터미널, Windows=PowerShell |
| **Node.js** | Codex 같은 도구를 돌리기 위한 실행 환경(엔진) |
| **npm** | Node 도구를 설치하는 설치 관리자. `npm install -g`는 컴퓨터 전체에 설치 |
| **Codex (CLI)** | 터미널에서 대화하며 일을 시키는 **AI 코딩 에이전트** |
| **에이전트(Agent)** | 스스로 파일을 읽고·만들고·명령을 실행하는 AI |
| **Git** | 파일 변경 기록·되돌리기 도구 |
| **GitHub** | Git 저장소를 인터넷에 올려두는 곳 (코드의 클라우드) |
| **repo(저장소, repository)** | 프로젝트 파일이 담긴 폴더 묶음 |
| **clone(클론)** | GitHub의 저장소를 내 컴퓨터로 복제해 받기 |
| **Markdown / .md** | 기호로 제목·표를 표현하는 가벼운 문서 형식 |
| **DOCX** | MS Word 문서 형식 (우리 산출물 최종 형태) |
| **RAG** | AI가 지어내지 않게 **신뢰 자료를 먼저 찾아 답하게** 하는 방식 |

## 🤖 Codex 운영 용어
| 용어 | 쉽게 말하면 |
|---|---|
| **프롬프트(prompt)** | AI에게 주는 요청/지시문 |
| **harness(하네스)** | AI가 매번 헤매지 않게 도구·규칙·절차를 묶어둔 작업대 |
| **AGENTS.md** | 그 폴더에서 AI가 따라야 할 규칙을 적어둔 지침 파일 |
| **skill(스킬)** | 반복 작업을 정해둔 절차 묶음 (예: BMC 스킬) |
| **context / token** | AI가 한 번에 보는 정보(맥락). token은 그 정보의 처리 단위 |
| **effort / fast** | AI가 얼마나 깊게(비싸게) 생각할지 조절. 가벼운 일=fast |
| **`[가정]`** | 근거 없이 일단 넣은 값이라는 표시 (나중에 검증 필요) |

## 📈 창업 용어
| 용어 | 쉽게 말하면 |
|---|---|
| **MVP** (Minimum Viable Product) | 검증에 꼭 필요한 **최소 기능 제품** |
| **BMC** (Business Model Canvas) | 사업 구조를 9칸으로 정리한 한 장 |
| **GTM** (Go-To-Market) | 어떻게 시장·첫 고객에게 들어갈지 전략 |
| **퍼널(funnel)** | 고객이 인지→관심→구매→재구매로 좁아지는 깔때기 |
| **CAC** | 고객 1명 데려오는 비용 (Customer Acquisition Cost) |
| **LTV** | 고객 1명이 평생 주는 매출 (Lifetime Value). LTV/CAC≥3이면 양호 |
| **TAM / SAM / SOM** | 전체 시장 / 도달 가능 시장 / 1년 내 현실 목표 시장 |
| **unit economics(단위 경제성)** | 고객 1명당 벌고 쓰는 돈이 남는지 |
| **WTP** | 지불 의향 (얼마면 낼지) |
| **페르소나** | 가상의 대표 고객 1명을 구체화한 것 |
| **레드팀 / VC 리뷰** | 투자자처럼 약점을 일부러 공격해 점검 |
| **합성 소비자** | AI로 만든 가상 고객 패널 (가설 점검용, 진짜 검증 아님) |
| **피어 리뷰(peer review)** | 동료끼리 서로의 산출물을 점검·피드백 |
| **피치(pitch)** | 아이디어를 짧게 설득력 있게 발표하는 것 |
