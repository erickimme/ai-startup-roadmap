# 05. 개발 로드맵 — 건강이음

## MVP 범위
- **포함**: 모국어 채팅 → 증상/진료과 안내 + 출처 표시 + 병원 찾기(공공 API) + 방문 준비 체크리스트. 언어 2종(베트남어·중국어) 우선.
- **제외**: 진단·처방, 예약 연동, 보험 청구, 음성, 앱(웹 먼저).

## 핵심 사용자 흐름
1. 언어 선택 → 2. 증상/상황 입력 → 3. 진료과·준비물 안내(+출처, "진단 아님" 고지) → 4. 주변 병원/보건소 링크

## 기술 스택
| 레이어 | 선택 | 이유 |
|---|---|---|
| 프론트 | Next.js(웹, 모바일 최적화) | 설치 장벽 0 |
| 백엔드 | Supabase | 빠른 인증·DB |
| AI | LLM API + **RAG(검증 의료 콘텐츠)** | 출처 기반·환각 억제 |
| 데이터 | 공공보건 포털·병원 찾기 API | 신뢰 출처 |

## 12주 마일스톤
| 주차 | 마일스톤 | 검증 기준 |
|---|---|---|
| 1-4 | RAG 콘텐츠셋 v1 + 채팅 | 의료진 감수 통과율 |
| 5-8 | 2개 언어 + 병원찾기 | 파일럿 사용 |
| 9-12 | 안전가드(응급·진단금지) 강화 | 오답·위험응답 0 목표 |

## 기술 의사결정
- Build vs Buy: 콘텐츠는 **공신력 출처 큐레이션**(자체 생성 X)
- 수작업으로 둘 것: 초기 콘텐츠 의료 감수
- 기술 부채로 받아들일 것: 언어 2종부터(확장은 나중)

## 보안/개인정보 (민감정보 — 최우선)
- 수집 정보: 증상 텍스트(가능한 비식별), 언어. **건강정보는 민감정보**
- 저장 방식: 최소 수집·암호화, 대화 기본 비저장 옵션
- 접근 권한: 역할 분리, 감사 로그
- 삭제/보관: 기본 단기 보관·즉시 삭제 요청 지원, 동의 고지

## 가장 약한 가정
- RAG 출처만으로 다양한 증상 질문에 **안전·정확**하게 답할 수 있을 것 `[가정]`
